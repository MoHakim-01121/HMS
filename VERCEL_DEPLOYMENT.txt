# Deploy Django ke Vercel

## Langkah-langkah:

### 1. Install Vercel CLI (optional)
npm install -g vercel

### 2. Login ke Vercel
vercel login

### 3. Deploy
vercel

Atau push ke GitHub dan connect di Vercel Dashboard.

### 4. Set Environment Variables di Vercel Dashboard
Settings > Environment Variables:

SECRET_KEY=your-random-50-character-secret-key
DEBUG=False
ALLOWED_HOSTS=.vercel.app,your-custom-domain.com
DATABASE_URL=postgresql://...

### 5. Database Options untuk Vercel:

a) Vercel Postgres (recommended):
   - Install: vercel postgres create
   - Auto-configured dengan DATABASE_URL

b) External PostgreSQL:
   - Supabase (free tier)
   - Neon (serverless PostgreSQL)
   - Railway
   - ElephantSQL

### 6. Static Files
Whitenoise akan handle static files automatically.

### 7. Media Files (User Uploads)
Untuk production, gunakan:
- Vercel Blob Storage
- AWS S3
- Cloudinary

### 8. Cek Deployment
- Visit: your-app.vercel.app
- Check logs: vercel logs

## Files yang sudah disiapkan:
✅ vercel.json - Konfigurasi Vercel
✅ build.sh - Build script
✅ requirements.txt - Updated dengan dependencies
✅ vercel_wsgi.py - WSGI handler untuk Vercel
✅ settings.py - Whitenoise middleware

## Security Checklist:
✅ SECRET_KEY dari environment
✅ DEBUG=False
✅ ALLOWED_HOSTS restricted
✅ HTTPS auto-enabled di Vercel
✅ Static files dengan Whitenoise

## Commands:
# Deploy
vercel --prod

# Check logs
vercel logs

# Set env variable
vercel env add SECRET_KEY

## Troubleshooting:
1. Build fails: Check requirements.txt versions
2. Static files 404: Run collectstatic in build.sh
3. Database error: Set DATABASE_URL in Vercel
4. 500 error: Check vercel logs

## Biaya:
- Hobby plan: Free (enough untuk start)
- Pro plan: $20/month (untuk production)

## Limitations di Free Tier:
- 100GB bandwidth
- Serverless function timeout: 10s
- Lambda size: 50MB

Jika butuh PDF generation (WeasyPrint), consider:
- Vercel Pro (larger lambda)
- Background workers (Celery + Redis)
- External API untuk PDF generation
